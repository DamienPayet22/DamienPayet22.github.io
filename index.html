SAE 101 : Raising awareness of computer hygiene and cybersecurity

In this SAE, we discovered the concept of cybersecurity, became aware of the many dangers of the Internet thanks to the ANSI MOOC, and learned how to protect our personal data. During our practical work, we discovered different types of cyber-attack, such as shoulder surfing, keyloggers and social engineering. We also experimented with a cyber-attack on our own virtual machines, using commands like “fcrackzip” to guess a password.

SAE 102 : Learn about computer networks

In this SAE, our aim was to send data, acquired by sensors and connected to a raspberry pi4 board, to a broker. To achieve our goal, we had to go through several steps. The 1st step was to find the raspberry's IP address so that we could connect to it via SSH. To do this, we used wireshark to find the frame containing the raspberry's IP address. The 2nd step was to write a program in Python to acquire data from the temperature and humidity sensor and switch on an LED. The 3rd step was to write a line in this program to send the sensor data to the broker.

SAE 105 : Data processing

In this SAE, our aim was to reproduce the Battleship game on a command prompt. To achieve this, we had to use and modify a C-language program containing the Battleship game, a program containing the client's source code and a program containing the server's source code. We then had to create header files so that the client could use the functions of the Battleship program to send data to the server, corresponding to the player's shots. The server would then receive this data and modify the board according to the client's shots. Then, after modifying the board, the server would in turn send its data to the client, so that it could see the changes to the board and find out whether or not it had succeeded in hitting a ship.

SAE 204 : Integrative project

In this SAE, our aim was to design a network using several network services and different ways of configuring network equipment. To achieve this, we chose to simulate the network using GNS3 software. We then had to use different equipment such as switches, routers, client machines and servers. We then configured a Windows server to act as a DNS server. Using the Windows server, we configured Active Directory, created users and assigned client machines to the domain. Next, we had to use a pfSense router in which we created sub-interfaces, rules enabling communication in the network. Finally, we used Cisco switches to create VLans. This brings us to inter-vlan routing.

SAE 301 : Implementing a transmission system

In this SAE, our objective was to design a transmission system to receive a message we have transmitted. To achieve this goal. We used several tools. First, we used Matlab software to build the transmission system. Next, we used Adalm-Pluto transmission devices to transmit and receive our message. Then, using the Matlab library, we were able to use blocks to create a transmission system. These blocks can be used to modulate and demodulate frequencies, filter the desired signal, and so on.

SAE 401 : Deploy operator infrastructure and customer services

In this SAE, our aim was to create a network architecture to represent an MPLS backone. This architecture is broken down into 3 parts. The 1st part of this network represents the MPLS backbone, which is made up of several Provider routers and 2 Provider Edge routers. These Provider Edge routers enable customer communications to be routed to their head office, without these customers being able to communicate with the MPLS backbone equipment. The 2nd part of the network represents customers who use the MPLS backbone to communicate with their head office. Finally, the 3rd part of the network represents the PPPoE link, which carries customer traffic to the Internet.
